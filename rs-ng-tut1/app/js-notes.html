  • Function Invocation and Execution Stack
    • Invocation - Running a function - In Javascript , by using parenthesis ()
    • Execution stack - Every time a function is invoke a execution context is created in the stack...
      function fn2(){ console.log('fn2');}
      function fn1(){ fn2();}
      fn1();       		This create a three execution contexts in the stack , in following order
      • Execution context for fn2()
      • Execution context for fn1()
      • Global Execution context and this
      		Stack Top -> |fn2 execution context | fn1 execution context | Global execution context |
  • Functions , Context and Variables Environment
    • Variable Environment
      function fn2(){ var myvar ; console.log(myVar);}
      function fn1(){ var myvar = 2 ; console.log(myVar); fn2();}
      var myVar = 1 ;
      fn1();
      console.log(myVar);       		stack top --> | fn2 context --> myVar is undefined |
      | fn1 context --> myVar = 2 |
      | Global context --> myVar = 1 |
  • Scope Chain - Understand
    function fn2(){ console.log(myVar);}
    function fn1(){ var myvar = 2 ; fn2();}
    var myVar = 1 ;
    fn1();
    		stack top --> | fn2 context --> |
    | fn1 context --> myVar = 2 |
    | Global context --> myVar = 1 |
    		Function Execution context has reference to it outer environment Here fn2 has reference to it outer context i.e Global Context because fn2 sits lexically on the top of Global environment so it gets the myVar value as 1.
